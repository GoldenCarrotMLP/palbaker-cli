To avoid manually wiring up loading spinners and disabling buttons for every single event handler now and in the future, the most elegant, scalable, and DRY (Don't Repeat Yourself) solution is to use a **Python decorator pattern** combined with Flet's native event-handling model.

When a user clicks a button in Flet, the event object (`e`) automatically carries a reference to the triggering element in `e.control`. By leveraging this, you can write a single, universal decorator that intercepts any CLI action, handles all the loading states and button locking automatically, and restores the UI when the task completes.

---

## 🛠️ The Solution: The `@cli_action` Decorator

Create a new utility file `ui_client/utils.py` to hold this decorator. It will automatically:
1.  Locate and show the View's `self.refresh_spinner`.
2.  Disable the clicked button (`e.control`) so the user cannot double-click.
3.  Optionally change the button's icon or append `...` to indicate it is actively working.
4.  Execute the actual async CLI task.
5.  Safely restore the button and hide the spinner in a `finally` block—even if the CLI throws an error or crashes.

### 1. Create `ui_client/utils.py`

```python
# ui_client/utils.py
import functools
import flet as ft  # type: ignore

def cli_action(func):
    """
    Universal decorator for Flet View event handlers.
    Automatically manages button disabling, icon indicators, and global spinners.
    """
    @functools.wraps(func)
    async def wrapper(self, e, *args, **kwargs):
        # 1. Identify the view's global spinner
        spinner = getattr(self, "refresh_spinner", None)
        if spinner:
            spinner.visible = True
            try:
                spinner.update()
            except Exception:
                pass

        # 2. Identify the clicked button/control
        control = getattr(e, "control", None)
        orig_disabled = False
        orig_icon = None
        orig_text = None

        if control:
            orig_disabled = getattr(control, "disabled", False)
            control.disabled = True
            
            # Visual feedback: Change button icon to hourglass/sync while working
            if hasattr(control, "icon"):
                orig_icon = control.icon
                control.icon = ft.Icons.HOURGLASS_EMPTY_ROUNDED
                
            # Visual feedback: If the button has an inner text object, we can note it
            if hasattr(control, "text") and isinstance(control.text, str):
                orig_text = control.text
                
            try:
                control.update()
            except Exception:
                pass

        try:
            # 3. Execute the actual CLI-bound View method
            return await func(self, e, *args, **kwargs)
            
        finally:
            # 4. Safely restore the button state
            if control:
                control.disabled = orig_disabled
                if orig_icon is not None:
                    control.icon = orig_icon
                if orig_text is not None:
                    control.text = orig_text
                try:
                    control.update()
                except Exception:
                    pass

            # 5. Hide the global spinner
            if spinner:
                spinner.visible = False
                try:
                    spinner.update()
                except Exception:
                    pass

    return wrapper
```

## 💻 How to Use It in Your Views

Because Flet supports native `async def` event handlers, you can decorate your view methods directly. You no longer need to write boilerplate logic for buttons.

### Example A: Integrating with `SettingsView`

Simplify your handlers in `views/settings_view.py`:

```python
# views/settings_view.py
import flet as ft  # type: ignore
from ui_client.utils import cli_action

class SettingsView:
    def __init__(self, page: ft.Page, settings: dict, on_rebuild_db_callback):
        # ... [Your picker setups remain identical] ...
        
        # Ensure every view has a unified spinner control
        self.refresh_spinner = ft.ProgressRing(width=16, height=16, stroke_width=2, visible=False)
        
        self.view = ft.Column(
            scroll=ft.ScrollMode.AUTO,
            controls=[
                # ...
                ft.Row([self.refresh_spinner, ft.Text("Settings Management")]),
                # ...
            ]
        )

    # Decorate the handler. Button locks and spinners are handled automatically!
    @cli_action
    async def _async_verify(self, e):
        result = await self.cli.env_verify()
        if result.get("status") == "success":
            self.show_snackbar("Verification completed successfully!", ft.Colors.GREEN)
        else:
            self.show_snackbar(f"Verification failed: {result.get('message')}", ft.Colors.RED)

    @cli_action
    async def _async_manage_ue4ss_with_action(self, e, action: str):
        # ... [Your action logic here] ...
        pass
```

### Example B: Integrating with `CreatorView`

Add a spinner and decorate your actions in `views/creator_view.py`:

```python
# views/creator_view.py
import flet as ft  # type: ignore
from ui_client.utils import cli_action

class CreatorView:
    def __init__(self, page: ft.Page, settings: dict):
        # ... [Setups] ...

        self.view = ft.Column(
            expand=True,
            controls=[
                ft.Row([

                    self.add_pal_btn
                ]),
                # ...

    @cli_action
    async def _async_create_pal(self, e, pal_id: str, template_id: str):

            self.show_snackbar(f"Created Pal: {pal_id}", ft.Colors.GREEN)
            await self._async_refresh_pals()
        else:
            self.show_snackbar(f"Error: {result.get('message')}", ft.Colors.RED)
```

## 🎯 Advantages of this Architecture
*   **0% Boilerplate:** Writing a new button handler in the future only requires adding `@cli_action` to the method signature.
*   **Centralized Diagnostics:** If you decide to change the loading icon, change how loaders are displayed globally, or add audio cues, you only need to modify **one file** (`ui_client/utils.py`).
*   **Failsafe Exception Handling:** The use of `try...finally` guarantees that if a CLI command hangs or throws a Python exception, the clicked button is always unlocked so the user does not get stuck.
