- **Headless CLI Decoupling & Crash Protection:**
  - Patched `palbaker_cli.py` to add `run_in_thread` inside `DummyView` to handle thread decoding tasks smoothly in headless operations without UI thread-spawning errors.
  - Implemented `sync=True` blocking across CLI-bound `altermatic` and `creator` commands (e.g. `add`, `delete`, `save` actions), converting asynchronous daemon threads to synchronous blocking operations to prevent premature exits and disk-write corruption.
  - Resolved `SettingsView` initialization errors by ensuring the constructor accepts and maps `on_save_callback` and triggers it inside a non-blocking update block.
  - Fixed an `AttributeError` crash on `altermatic save` due to missing `args.mod` inside the command handler by parsing `--data` JSON payload on-the-fly to extract the `CharacterID`.
  - Added a self-healing mapping inside the dispatcher (`run_pipeline_stream`) to automatically convert GUI action underscores to dashes (`action.replace("_", "-")`), fixing instant failures when triggering `create-blend`.

- **Visual Indicators and Indeterminate Progress Feedback:**
  - Added indeterminate visual loading indicators (`value=None`) for `Extract Pal` and `Decompile` actions on mod cards, locking controls during remote execution to prevent double-click actions.

- **Robust JSON Parsing for CLI Status Logs:**
  - Modified the dispatcher's `_execute` command scanner to scan stdout line-by-line from bottom to top, identifying and parsing the last matching JSON line `{...}` while ignoring pre-import status updates, stdout chatter, and Blender warning logs.

- **DRY CLI Decorator Utility Pattern:**
  - Created a universal `@cli_action` decorator inside `ui_client/utils.py` to decouple visual loading indicators and button disabling from event handlers natively, offering clean error fallback and zero boilerplate.

- **UI-Sourced Multi-Step Chaining:**
  - Refactored `_async_execute_pipeline` in `views/mods_view.py` to handle multi-step actions (`cook` and `full`) sequentially by chaining `cook -> pack` and `push -> cook -> pack` terminal executions using an `asyncio.Future` flow waiter.
  - Patched `build_mod.py` pack-stage files-found verification to allow successful exit for empty vanilla Altermatic packs when configuration or audio assets are otherwise successfully staged.
